Feature Explanations

The manual pages comprising this section describe all of the features that jSymbolic extracts, as well as important introductory information for each feature type. The features are divided into the following groups:

  • Overall pitch statistics
  • Melody + melodic intervals
  • Chords + vertical intervals
  • Rhythm
  • Instrumentation
  • Texture
  • Dynamics
  • N-gram sequences
  • MEI-specific features

These manual pages were originally drawn largely from Chapter 4 of Cory McKay's dissertation, although many improvements and clarifications have been made since then, and many new features have been added.

Note that, although many of the feature descriptions use MIDI-specific terminology, they are all compatible with alternative formats that jSymbolic can extract features from, namely MEI. Part of what jSymbolic does when encountering MEI files is to convert them to MIDI data in a very rich way, such that all the MIDI-related features can be extracted from them. MEI data that cannot be encoded as MIDI is maintained in a separate data pipeline, and features can be extracted from this pipeline as well (see the section on MEI-specific features).

If a feature cannot be calculated for whatever reason for a given piece of music, the default jSymbolic behavior is to save a value of -1 for one-dimensional features, and null for multi-dimensional features.
