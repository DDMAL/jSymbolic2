Feature Explanations: MEI-Specific Features

GENERAL NOTES

There is certain information that cannot be encoded in jSymbolic's native MIDI, due to inherent representational limitations of the MIDI format. jSymbolic uses a separate data pipeline to provide access to information of this kind that can instead be encoded in MEI, and jSymbolic can use "MEI-specific features," such as those described below, to extract useful information from this pipeline. In other words, jSymbolic uses MEI-specific features to access information that is available in MEI files, but cannot be encoded in MIDI. MEI-specific features should only be used when extracting features from a musical collection consisting exclusively of MEI files, as a feature set extracted from a mixture of MIDI and MEI files would be inconsistent if MEI-specific features are available for MEI instances but not for MIDI instances.

Note that jSymbolic can only parse MEI files conforming to the effective "standard" MEI encoding specification as it existed in 2017, when jSymbolic's MEI parser was implemented (see the introductory section of this manual for more details on this).

JSYMBOLIC FEATURE CATALOGUE (MEI-SPECIFIC FEATURES SECTION)

  • S-1 Number of Grace Notes: The total number of grace notes in a piece (i.e. the number of notes indicated as grace notes in the MEI encoding) divided by the total number of pitched notes in the music.
  • S-2 Number of Slurs: The total number of notes marked with slurs in the piece (i.e. the number of notes indicated as notes with slurs in the MEI encoding) divided by the total number of pitched notes in the music.
