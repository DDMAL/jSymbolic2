jSymbolic Version History

All updates stated below are since jSymbolic 1.0

Currently Being Worked On:

  • Go to the GitHub master branch page to see the most current stable version of the code currently being worked on. This is not the release version, however: the proper release version, which is recommended for research use, is available on the jMIR SourceForge page.
  • Go to the GitHub development branch page to see the cutting edge development version of the code. Of course, it is not recommended that you deploy and use this version directly, as it may not be fully tested and stable at any given time.

jSymbolic 2.2:

  • Extensive new HTML tutorial:
    • Including working files.
    • Accessible via GUI or on-line.
  • Optional new configuration file that automatically disables features that could potentially measure biased information in improperly constructed datasets.
    • e.g. disables tempo-related features, in case the dataset has tempo data linked to encoding source rather than musical content.
    • Designed to be especially useful for dealing with (Western) early music, as the disabled features are not relevant to this kind of music.
  • Minor bug fixes:
    • Changed the Chord Duration feature to reflect duration in quarter notes rather than in seconds.
    • Fixed typos in the hard-coded names of three features.
    • Fixed bug involving pieces with no pitched notes.
  • Updated the manual, README, code documentation and configuration files to reflect all changes.
    • Added minor documentation clarifications and corrections in general.

jSymbolic 2.1:

  • Features:
    • Added 74 new features.
    • Reordered, renumbered, renamed and/or reworded documentation of many features (new and old) for the purpose of clarity.
  • New reports / analysis:
    • Detailed formatted reports on the MIDI message-by-message contents of individual MIDI and (internally converted) MEI files.
    • Reports on the intra-consistency and inter-consistency of MIDI and MEI files. Useful for testing for dataset bias and for debugging during feature development.
  • User interfaces:
    • Added recursive directory parsing option to the GUI and command line.
    • Added file count reporting to the GUI for when files are removed from the processing queue.
    • Removed the indication of whether features are sequential from the GUI features table (in order to create more space).
  • Improved error handling:
    • Enhanced functionality for automatically validating and error-checking new features as they are added to jSymbolic.
    • Additional general new error detection functionality.
    • Improved clarity of error messages, removed redundant data.
    • Added miscellaneous internal debugging methods.
  • Adjusted reactions to unspecified data in symbolic music files:
    • Defaulted time signature to 4/4 if no time signature is specified in the MIDI file (this is the default behaviour for MIDI).
    • Defaulted tempo measurements to 120 BPM if no tempo change messages are specified in the MIDI (this is the MIDI default).
  • Improved character handling:
    • Updated Apache Xerces to version 2.11.0.
    • Set saving of ACE XML files to consistently use UTF-8 encoding.
    • Improved error messages to be more informative when encountering invalid characters in file names.
  • Miscellaneous additional internal additions and changes.
  • Miscellaneous minor bug fixes.
  • Updated the manual, README, code documentation and configuration files to reflect all changes.
    • Added minor documentation clarifications and corrections in general.
  • Updated the ACE and UtilityClasses support libraries to the newest (improved) versions.

jSymbolic 2.0:

  • Extensive expansion and improvement of the jSymbolic features and their support infrastructure:
    • Implemented 49 features that had been proposed in jSymbolic 1's support literature, but which had not yet been implemented.
      • This includes all the features related to chords and vertical intervals, among others.
    • Designed and implemented 12 entirely new features.
    • Updated and improved a number of the previously implemented 111 features found in jSymbolic 1.
    • Added the ability to implement and extract specialized MEI-specific features (i.e. features that cannot be extracted from MIDI files).
      • An MEIFeatureExtractor architecture similar to the MIDIFeatureExtractor has been added in order to accomplish this.
      • More information can be found at Extending Software page.
    • Added numerous helper classes and methods related to features.
  • Added the ability to extract features separately and individually from multiple windows in a file (as opposed to only being able to extract features from a file overall, which was the only option in jSymbolic 1).
    • This includes the ability to use windows with or without offsets.
  • Added the ability to extract features from MEI files using our new custom-built jMei2Midi project (which can convert MEI files to MIDI), and which in turn uses the jMEI library.
  • Extensive interface and documentation improvements:
    • Added a detailed HTML manual.
    • Added an easy-to-use and well-documented dedicated API so that jSymbolic can be incorporated into external software without needing to resort to command line calls.
    • Expanded and redesigned the GUI in order to make it more powerful and easier to use.
    • Completely overhauled the command line interface design in order to enable access to more functionality and to make it more easily extensible.
    • Implementation of a new Rodan workflow for using jSymbolic in a distributed processing context.
  • Added a new configuration file format to facilitate the saving and reuse of settings, as well as to facilitate batch processing still further.
    • Configuration files can be used with the GUI, command line interface and API.
    • Configuration files can be edited directly with a text editor, or savings entered on the GUI may be saved to a configuration file using the GUI.
    • The configuration file format is designed to itself be easily extensible.
  • Extracted features may now be saved as basic CSV text files or as Weka ARFF files (ACE XML was the only output option in jSymbolic 1).
  • Substantially expanded, improved and streamlined error handling.
    • No longer terminates execution when problems occur, and produces log files to keep track of errors encountered during batch processing.
    • More error messages with more detailed and specific error explanations.
  • Added substantial unit testing.
  • Cleaned up code and documentation in general.
  • Numerous miscellaneous bug fixes.
  • Updated the ACE and UtilityClasses support libraries to the newest (improved) versions.

jSymbolic 1.2.1:

  • Added a command line interface to complement the already-implemented GUI. This involved updating the FeatureSelectionPanel class with a new public getAllAvailableFeatureExtractors method to provide access to the available features, as well as updating the private populateFeatureExtractors method to now use this new method to set up features for the GUI internally. New features that were previously added using the populateFeatureExtractors method should now be added with the getAllAvailableFeatureExtractors method.
  • Updated support libraries (ACE and UtilityClasses).

jSymbolic 1.2:

  • Updated support libraries (ACE and UtilityClasses).

jSymbolic 1.1:

  • Imported jSymbolic into the NetBeans framework.
