BASIC BACKGROUND

  • "jSymbolic" is a piece of software that extracts "features" from symbolic music files
    • Each feature represents some numerical aspect of the music
    • Each feature is a single statistical value
      • e.g. the proportionate amount of parallel motion in a score is a feature
    • jSymbolic is part of my larger "jMIR" framework
      • jMIR also includes components for extracting features from audio, lyrics, cultural data and more
  • "Symbolic music files" are computer-readable files that store instructions on how music should be played
    • Digital scores
    • No audio (although they can be synthesized)
    • Finale, Sibelius, MIDI, MEI, Music XML, etc.
  • "Weka" is a piece of software from the University of Waikato for doing machine learning and feature analysis
    • "Machine learning" involves automatically "teaching" computers to recognize patterns in data
    • Machine learning typically needs to be "trained" on "training data"
      • This "training data", in this case, consists of jSymbolic features extracted from symbolic music files
      • The more data you have, the better the results will likely be (at least as a first approximation)
